Introduction
Qu'est-ce que le web?
Internet
  • Réseau de réseaux
  • Utilise le protocole TCP/IP, par le port 80
  • Arpanet : années 70
Schéma
Le protocole HTTP
Présentation
  • Hyper Text Transfert Protocol - Tim Berners Lee
  • Adapté au multimédia
  • Léger, rapide, coût d'exploitation très bas
  • Dérive du protocole FTP
Le protocole HTTP
Requêtes
  • GET : demande quelque chose
  • POST : envoi de paramêtres
  • HEAD : demande les informations du document
  • PUT, DELETE, ... : utilisés par des outils spécifiques
Des clients, des serveurs
2 acteurs
  • Le client : il effectue des requêtes vers le serveur (ex: Firefox)
  • Le serveur : il exécute les requêtes et renvoie le résultat au client (ex: Apache)
Des clients, des serveurs
Rôle du client
  • Traduit les ordres en requêtes selon un protocole d'échange avec le serveur
  • Dialogue avec le serveur
  • Met en forme la réponse et la présente à l'utilisateur
Des clients, des serveurs
Rôle du serveur
  • Transforme l'url en fichier ou en script
  • Vérification d'identité
    • L'utilisateur est-il bien celui qu'il prétend être ?
  • Vérification d'accès
    • L'utilisateur a-t-il le droit de voir la ressource ?
  • Envoi de la réponse au client
  • Suivi de fichier logs
  • Traitements évolués (interprètes PhP, perl, accès aux BDD, etc.)
Des clients, des serveurs
Exemples de serveurs
  • Apache
  • Jigsaw
  • MacHTTP
  • Microsoft Server
  • Netscape server
  • ...
Fonctionnalités des serveurs
Services rendus
  • Envoi une page en ajoutant les en-têtes appropriés
  • Exécute un script et envoie le résultat sous forme de page (formulaires, PhP)
  • Exécute un script et envoie le résultat tel quel (fonctionnement AJAX)
Fonctionnalités des serveurs
Possibilités
  • Vérifie la machine d'origine (ou dernier proxy)
  • Vérifie l'accès (login/password)
  • Construit une page à la volée avec des ordres spécifiques
Fonctionnalités des serveurs
Possibilités
  • Reconnaître les entêtes de requêtes et servir des pages différentes suivant le client ou ses préférences
    • Avoir des pages différentes selon le browser, la langue
  • Encrypter les données et sécuriser les transactions
Historique

HTML
HTML
Présentation
  • Langage de structuration de document basé sur une DTD (Document Type Definition)
  • Défini par des balises interprétées par un navigateur
  • Permet de décrire des pages et d’en contrôler l’apparence
  • Résultat pouvant varier suivant la configuration du poste du client
HTML 4.01
Présentation
  • Internationalisation (meilleure gestion des langues)
  • Accessibilité (meilleur support d'agents non graphiques)
  • Nouveau modèle de tableaux (meilleur contrôle de la structure et du formatage)
  • Documents composés (incorporation d'objets média et d'applications génériques)
HTML 4.01
Présentation
  • Feuilles de style (séparation structure/présentation)
  • Scripting (pages Web dynamiques/interactifs)
  • Impression (collections de documents)

Éléments et Attibuts
Principe de balisage
  • Défini par : étiquette de début, contenu, étiquette de
    • <etiquette> contenu </etiquette>
  • Contenu vide : pas d'étiquette de fin
    • <br/> ou <img .... />
  • Attention !! étiquettes optionnelles, insensibles à la casse -> de la rigueur!
Éléments et Attibuts
Attributs
  • Propriétés associées aux étiquettes
  • Définis par un nom et une valeur
    • <img id="logo" src="logo.gif" alt="Cliquez ici"/>
Syntaxe
Ce qu'impose la norme :
  • Balises en minuscule
  • Attributs entre simples ou doubles quotes
  • Tout attribut attribut doit avoir une valeur
  • Toute balise doit être fermée
  • Les imbrications doivent être respectées
  • Exemples :
    • <input type="checkbox" checked="checked" />
    • <p class="gras"> du contenu.... </p>
    • <p><ul> <li>item1</li> <li>item2</li> </ul></p>
XHTML1.0
XHTML1.0
Présentation
  • Extensible, dérive du XML
  • Portable, compatible HTML et XML
  • Conserve les principes de HTML4.0
  • Normé par le W3C
    • http://validator.w3.org
XHTML1.0
Conformité
  • Respect de la norme
  • L'élément racine html doit contenir un attribut normatif xmlns
  • Balise DOCTYPE obligatoire, faisant référence à un fichier normatif du W3C
    • <!DOCTYPE html PUBLIC "-//W3C//DTD XHTML 1.0 Strict//EN"
      "http://www.w3.org/TR/xhtml1/DTD/xhtml1-strict.dtd">
    • <!DOCTYPE html PUBLIC "-//W3C//DTD XHTML 1.0 Transitional//EN"
      "http://www.w3.org/TR/xhtml1/DTD/xhtml1-transitional.dtd">
XHTML1.0
Conformité
  • Strict : exclut des éléments et attributs de "mise en forme".
    Ils sont décrits dans les feuilles de style CSS
  • Transitional : étend en acceptant certaines balises de mise en forme
Exemple

Les balises META
Pourquoi ?
  • Guider les moteurs de recherche
  • "Influencer" les navigateurs
  • "Commander" les robots
Les balises META
Comment ?
  • Situées entre les balises head
  • <meta [attribut1="valeur1"] [attribut2="valeur2"]... />
  • Différentes informations :
    • sur vous, vos pages
    • pour les revisions, les rafraîchissements de pages
    • et bien d'autres!
Les balises META
Auteur
  • Permet de vous référencer sur un moteur de recherche
  • <meta name="author" lang="fr" content="Peio Loubière" />
  • name : nom de la balise META
  • lang : langue utilisée
  • content : auteur de la page
Les balises META
Mots clés
  • Précise les mots clés du site
  • <meta name="keywords" content="liste de mots clés" />
    • <meta name="keywords" content="EISTI,HTML,XHTML,CSS,CPI,AREL,cours" />
Les balises META
Robots
  • Demande aux robots de réindexer la page tous les n jours
    • <meta name="revisit-after" content="n" />
  • Orienter les robots
    • <meta name="robots" content="combinaison souhaitée" />
  • La combinaison se détermine selon les valeurs suivantes :
    • all : indexe toutes les pages du site
    • none : n'indexe aucune page
    • (no)index : (n')indexe (pas) la page en cours
    • (no)follow : (n')indexe (pas) les liens
  • <meta name="robots" content="index,nofollow" />
Les balises META
Instructions au navigateur
  • Permet de définir le jeu de caractères d'affichage
    • <meta http-equiv="content-type" content="text/html;charset=ISO-8859-1" />
  • Demande au navigateur de rafraîchir la page toutes les n secondes
    • <meta http-equiv="refresh" content="n" />
  • Demande au navigateur de ne pas utiliser le cache
    • <meta http-equiv="pragma" content="no-cache" />

Balises
Utilisation de balise
  • Nécessité d'utiliser les bonnes balises
  • Caractérisation par 2 attributs
    • id : caractérise une balise de manière unique
    • class : caractérise un ensemble de balises
id et class
Différences
  • Attribut id
    • permet de remplacer une ancre
    • formalisme CSS : #nomId
  • Attribut class
    • formalisme CSS : .nomClass
    • permet de cumuler des mises en forme :
      <p class="pbleu gras">
Imbrication
/*  cette classe ne s'applique que dans une balise span contenue dans une balise p  */
p span{
color: red;
font-size: 22px;
}
  • Attention à l'ordre des balises
  • L'imbrication de balises div afin d'appliquer différentes classes est inutile, préférer la surcharge de classe
  • Une balise imbriquée dans une balise parente hérite de la mise en forme de cette dernière, à l'exception des marges, bordures et quelques autres
Balises div et span
Présentation
  • Balises "muettes" de mise en forme
  • Permet de disposer les éléments dans la page
  • Utilisation obligatoire de ces balises -> PAS DE TABLEAU
  • Différenciation balises ligne et bloc
Balises bloc et ligne
Deux groupes
  • Bloc
    • <p>,<ul>,<li>,<div>,<h1>,...
    • Positionnement : les uns au dessus des autres
  • Ligne
    • <a>,<span>,<img>,<em>,...
    • Positionnement : les uns à côté des autres
Balises bloc et ligne
Notions
  • Boite invisible configurable, autour du contenu
  • Ligne : prochaine boite se trace "à côté"
  • Bloc : prochaine boite se trace "en dessous"
  • Posiibilité de modifier le comportement
    • Un intrus ! display: (inline|block|none) ;
    • visibility: (visible|hidden)Un autre intrus;
Configuration
Dimensions
  • height, width : dimensions de la boite
  • border : bordure de la boite
  • overflow : dépassement du contenu de la boite
    • visible : par défaut, la boite s'élargit
    • hidden : masque le dépassement
    • scroll : crée des barres de défilement
    • auto : crée des barres de défilement si nécessaire
Configuration
Marges
  • 2 types de marges
    • padding : marges intérieures, entre le contenu et le bord de la balise
    • margin : marges extérieures, entre deux bords de balises
.bloc{
 width: 100px;
 height: 100px;
 background-color: yellow;
 margin-left: 15px;
 margin-top: 200px;
}
Configuration
Positionnement fixe, relatif ou absolu
  • fixe : position identique même si l'on bouge les ascenceurs latéraux
  • relatif : position dépendante du père ou de sa position dans le flux d'affichage
  • absolu : bloc "positionné" à partir du coin haut-gauche de la fenêtre du navigateur
Configuration
Positionnement fixe, relatif ou absolu
  • Permet de positionner les blocs les uns par rapports aux autres
  • Dangereux : pas ou peu d'adaptation aux résolutions
  • Seul moyen de superposer les éléments (propriété z-index)
Configuration
Positionnement flottant
  • Propriété float, utilisée pour aligner des blocs les uns à côté des autres
  • Sort du flux courant, pas de calcul de hauteur
  • 2 types de flottaison : left, right
  • fin de flottaison : none
Pseudos formats
Présentation
  • Définis en CSS
  • Affectent la mise en forme sous conditions
  • Permettent d'ajouter de l'interactivité
  • S'utilisent sur une propriété ou une balise
Pseudos formats
Quelques exemples
  • :hover : lorsque la souris survole la balise
  • :active : lorsque l'on clique sur un lien
  • :visited : lorsqu'un lien a été visité

  • Et bien d'autres!
Pseudos formats
Un exemple : numérotation automatique de titres
  • Fonctionnalités de comptage fournies par CSS
    • counter-reset: cpt val : initialise le compteur cpt à val (par défaut 0)
    • counter-increment: cpt val : incrémente le compteur cpt de val (par défaut 1)
    • counter(cpt, style) : récupère la valeur du compteur cpt avec le style style (par défaut numérique, autres : roman, latin)
    • content: val : affiche la valeur val
Pseudos formats
h1:before {
    content: "Chapitre " counter(chapitre) ".";
    counter-increment: chapitre; /* Ajoute 1 au chapitre */
    counter-reset: section;      /* Remet la section à zéro */
}

h2:before {
    content: counter(chapitre) " - " counter(section) ".";
    counter-increment: section;
}
